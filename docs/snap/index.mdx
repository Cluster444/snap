Snap helps you build apps that can be used by people, AI agents, and other apps without rebuilding the same domain logic for every interface.

Instead of starting from a UI or a server route, you start with the capabilities your app should provide. Then you connect those capabilities to the clients that need them: a web app, an agent, a CLI, a mobile app, or another Snap App.

<Warning>
  Snap is currently a design-stage product. These docs describe the intended developer experience, not a stable public API.
</Warning>

## How Snap fits together

Snap starts with three ideas:

1. **Apps** are the products you run and deploy.
2. **Blocks** are reusable domain capabilities an App can use.
3. **Clients** are how humans, agents, or other apps interact with an App.

## Build your first app in a snap

Create an app, add the Blocks it needs, then run it locally:

```bash
snap create todos
snap use todos
snap dev
```

The App definition stays focused on what the app can do:

```ts
import { app } from "@snaplab/snap";
import { identity } from "@snaplab/core";
import { todos } from "@leanbod/todos";

export default app("todos", {
  blocks: [identity(), todos()]
});
```

From there, you can add a web Client for people, an agent Client that can help manage tasks, or another App that creates tasks from its own workflow.

For the first milestone, the goal is simple: create an App, connect a human Client, connect an agent Client, and deploy the result.
