Snap is a framework for building apps that humans, agents, and other apps can use together.

You build a Snap App by composing reusable Blocks. Then you create Clients that connect to the App: a web UI for humans, an agent client for AI systems, or another app that needs to use the App's capabilities.

<Warning>
  Snap is currently a design-stage product. These docs describe the intended developer experience, not a stable public API.
</Warning>

## The three ideas

Snap starts with three user-facing concepts:

1. **Apps** are the products you run and deploy.
2. **Blocks** are reusable domain capabilities an App can use.
3. **Clients** are how humans, agents, or other apps interact with an App.

Engine details such as Primitives, Transport, kernels, and deployment internals sit underneath those concepts. You do not need to learn them before building your first App.

## What the first App does

The first milestone is a chat App:

1. Create a Snap App.
2. Install starter Blocks such as identity and chat.
3. Add a web Client for a human.
4. Add an agent Client that can join the same room.
5. Run the App locally.
6. Deploy the App and its configured Clients.

That path proves the core idea without introducing factories, orchestration, registries, or hosted control-plane features.

## CLI shape

The intended CLI keeps the common App path first-class:

```bash
snap create <app>
snap use <app>
snap dev [app]
snap deploy [app]
snap modify [app]
```

Blocks and Clients have their own commands when you need to add or create them:

```bash
snap block create <name>
snap block install <name>

snap client create <type>
```
